kotlin 类与接口

发表于 2020-12-26 分类于 kotlin
描述kotlin种的面向对象编程

interface

​ 与java一样，kotlin也是通过interface关键字来声明一个接口，同时java8过后接口里面可以定义defalute方法和变量，并且变量可以进行初始化，但是在kotlin中，变量不能直接赋值，但是可以通过get()方法进行赋值，如果没有在接口中对变量进行初始化，那么实现类必须对其进行初始化

class

​ 与java一样，kotlin也是通过关键字class来声明一个类，同时在class种也可以生面变量和方法。同时，在kotlin中没有new 关键字，直接调用构造函数即可

构造器

​ 在java创建对象时，都是通过调用构造器，有时候会存在多个构造器重载，而过多的重载方法不仅会造成代码冗余，而且不同构造器里面面的代码几乎很多重复，所以kotlin在这一点上做了优化，支持在构造器中添加默认值

如果一个非抽象类没有声明任何（主或次）构造函数，它会有一个生成的不带参数的主构造函数。

在 JVM 上，如果主构造函数的所有的参数都有默认值，编译器会生成 一个额外的无参构造函数，它将使用默认值。

init{}

  • ​ 上面多次出现了inti{},在kotlin中，主构造器里面只能包含参数，不能包含具体实现，跟java不一样，所以才有了init{},可以像需要在主构造器里面执行的操作放到这个代码块中，而且，当主构造器中的参数没有用var/val修饰时，他们只能在init块中和属性初始化时使用
  • class中可以声明多个init{},他们会根据在类中出现的先后顺序执行，这一点参考java的static变量和代码块，是一样的执行顺序

次构造器

  • kotlin中也支持声明多个构造器，但是只能由一个主构造器，可以有多个次构造器，但他们都必须直接或间接委托给主构造器，委托到同一个class中的其他构造器使用this,并且所有的次构造器都必须加上constructor关键字

  • 次构造器可以有代码块，但是他会在init{}之后执行，因为init{}实质上是属于主构造器的，即使该类没有主构造函数，这种委托仍会隐式发生，并且仍会执行init{}

成员变量

​ 在kotlin的class中，用var或val声明一个引用可变或不可变的变量，而且，除非显示指定延迟初始化，否则多有的变量都必须进行初始化，这一点和java不一样，在java的class中可以不用显示进行初始化，因为在类加载的时候jvm已经默认帮我们初始化过了

lateinit

​ 他是kotlin中的一个修饰变量的关键字，表示延迟初始化，被他声明的变量可以不用立即进行初始化

  • 他不能修饰基础数据类型(和java一样的8大基本数据类型)，可以修饰String和引用数据类型
  • 只能修饰var变量
  • 只能修饰不可空变量，即不能加?

by lazy

​ 他也是kotlin中的用来修饰变量的，也表示延迟初始化，但是他只能修饰val变量，当首次被调用时就会赋值，他后面跟的时一个lambda表达式

成员方法

  • ​ kotlin种的所有成员方法，默认都是public的，即可以全局访问，这一点跟java也不一样，java的默认作用域是包作用域
  • 同时方法默认都是final不可继承，如果希望可以被继承，需要添加关键字open

data class

​ 在java中需要一个javabean的时候，需要给每个方法设置get/set方法以及重写一些equal，toString等方法，代码非常繁琐，在kotlin中只需要在class前用一个data修饰即可

  • copy
  • componentN,N代表属性的索引位置

强制：数据类必须包含至少有一个参数的主构造器，且都必须用var/val修饰

object class

​ 表示一个单例class实例，在jvm加载的时候就会初始化，而且只初始化一次，天然的单例

companion object

​ 伴生对象，取代了static,只能在一个class的内部声明，它属于这个class,在jvm加载的时候就会初始化他，跟ststic起着类似的作用
